Dynamic Models: From Forces to Flight

An interactive guide to the physics-based and data-driven models for aerial vehicle dynamics.

Hover Computation Graph

The core of the simulation is the hover computation graph, which takes motor commands and computes the next state of the vehicle.

Motor Command ωᵢInitial input from the controller.
Motor/ESC DynamicsModels electrical to mechanical power conversion.
Rotor AerodynamicsCalculates thrust and torque from rotor speed.
Force/Torque AggregatorSums forces and torques from all rotors.
Newton-Euler Rigid BodyComputes vehicle acceleration from forces.
Integrator (RK4/Euler)Updates vehicle state over a time step.
Next StateThe new position, velocity, and attitude.

Terminology & Notation

TermDescription
RK4Classical fourth-order Runge-Kutta integrator for solving differential equations.
T = k_t ω²Simplified rotor thrust law, where T is thrust, ω is rotor speed, and k_t is a coefficient.
Q = k_q ω²Simplified reaction torque law.
ξ = (x,y,z)Position of the vehicle center of mass in the inertial frame.
η = (ψ, θ, φ)Yaw, pitch, roll Euler angles describing the attitude.
m·ξ̈ = -mgD + R·FNewton-Euler translational equation.

Rotational Inertia Quick Reference

ShapeMoment of Inertia
Solid cylinder/disk (about central axis)I_z = 0.5·m·r²
Rod about centerI = (1/12)·m·L²
Rod about endI = (1/3)·m·L²
Rectangular prismI_x = (1/12)·m·(b² + c²)

Try it Yourself!

Build and run the examples to see these models in action:

# Build the point mass example
cmake --build . --target point_mass_example
./build/point_mass_example

# Build the spring pendulum example
cmake --build . --target spring_pendulum_example
./build/spring_pendulum_example
